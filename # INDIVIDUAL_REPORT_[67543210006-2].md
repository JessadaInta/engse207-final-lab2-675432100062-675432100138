## ขอบเขตงานที่รับผิดชอบ
 User Service, Task Service, Docker Compose และการทดสอบระบบในส่วนที่ตนเองทำด้วย พร้อมทดสอบแบบ  E2E กับทีม

## สิ่งที่ได้ดำเนินการด้วยตนเอง
- การสร้าง  user-service ขึ้นมาเพื่อให้สามารถจัดการโปรไฟล์ รวมไปถึงการสร้างหน้าแสดงรายชื่อสำหรับ Admin ด้วย
- การทำ Docker compose โดยการเทสส่วนต่างๆ การเข้าถึง  Database การเช็ค Log ต่างๆ เพื่อหา Error
- การ Debug โค้ดในส่วนที่ทีม และตนเองได้ทำ รวมไปถึงการ Recap สิ่งที่ทำให้กันได้รู้ จะได้รู้ว่าอยู่จุดไหนกัน

## ปัญหาที่พบและวิธีการแก้ไข
1. HealtCheck เป็น UnHealty ซึ่งเป็นการเทสขั้นแรกของการรันระบบ ว่าสามารถใช้งานได้ไหม โดยเนื่องจากโค้ดที่ทำงานเกี่ยวกับ HealtCheck ไม่ทำงาน หรือไม่มี จึงได้ทำการแก้ไขให้ทำงานได้
2. การ Register และ ข้อมูลของ  auth_db ไม่เข้าไปในฐานข้อมูลของ user_db ทำให้เมื่อเรียกใช้ข้อมูลที่ register ไปแล้วไม่ได้ โดยใน user.js ขาดฟังก์ชันที่เรียกให้ข้อมูล Register ลงมาใน user_db ไม่มี จึงได้ทำการเพิ่มเข้าไป
3. การใช้ token ของ admin ไม่ได้ จึงเลือกการเจน hash ที่ตรงกันกับ bcryptjs เพราะตอนแรกเป็นแค่ bcrypt

## สิ่งที่ได้เรียนรู้จากงานนี้
การทำงานของ Database ทั้งที่แยกออกจากกันทั้งสามตัว ที่มีความสัมพันธ์กันอยู่ เช่นเมื่อ Register แล้วข้อมูลเข้าไปอยู่ในอีกฐานข้อมูลได้ยังไง เนื่องจากติดปัญหาตอนแรกจึงทำให้แก้นานมาก จนได้รู้การทำงานของมัน
